🧗 Warm-up Glossary — Vocabulary You Need Before §1

The Master Theorem uses dense mathematical notation. None of the underlying ideas are difficult — they just need careful introduction. If you are fluent in all of these already, skip straight to §1.

What is a recurrence relation? 📄 p. 79

A recurrence relation is a formula that defines the running time of an algorithm in terms of itself on a smaller input. Instead of calculating total work directly, we express T(n) — the time to solve a problem of size n — as a sum of: the time to solve smaller sub-problems, plus the extra work done outside those recursive calls.

Concrete example: Merge sort splits an array of n elements into two halves and merges them. Each half takes T(n/2) time. The merge step takes O(n) extra time. So: T(n) = 2·T(n/2) + n. This one equation fully describes merge sort's growth. 📄 p. 79

  T(n) = 2·T(n/2) + n

        ↑                ↑
  Two recursive calls  Extra work done
  each on half-size    by merge step
  input (n/2)          outside recursion

What does T(n) mean? 📄 p. 79

T(n) is a function that maps input size n to the running time of the algorithm. "T" stands for "Time." When we say T(n) = 2T(n/2) + n, we mean: the time to sort n items equals the time to sort n/2 items (done twice) plus n extra operations for the merge. 📄 p. 79

Base case: At some smallest input (like n = 1 element), T(1) = Θ(1) — the algorithm just returns immediately. Without a base case, the recurrence would recurse forever.

What does "divide and conquer" mean structurally? 📄 p. 79

Divide and conquer algorithms have a 3-part structure: Divide the problem into a smaller sub-problems, each of size n/b. Conquer by solving each sub-problem recursively (taking T(n/b) time each). Combine the results in f(n) extra time.

  Standard D&C recurrence form: T(n) = a·T(n/b) + f(n)

  a = number of sub-problems created at each level
  b = factor by which input size shrinks (each sub-problem is 1/b the size)
  f(n) = non-recursive work done at this level (divide + combine)

  Merge sort:   a=2, b=2, f(n)=n   →  T(n) = 2T(n/2) + n
  Binary search: a=1, b=2, f(n)=1  →  T(n) = T(n/2) + 1
  Karatsuba:    a=3, b=2, f(n)=n   →  T(n) = 3T(n/2) + O(n)
  Strassen:     a=7, b=2, f(n)=n²  →  T(n) = 7T(n/2) + O(n²)
SOURCE: 📄 pp. 79–90, 227–238

What is log base b — logb? 📄 p. 39

logb(x) asks: "to what power must I raise b to get x?" In other words, logb(x) = y means by = x. The course uses log₂ (log base 2) most frequently, often written just as "log." 📄 p. 39

  Examples (essential to memorize):

  log₂(1)  = 0        because  2⁰ = 1
  log₂(2)  = 1        because  2¹ = 2
  log₂(4)  = 2        because  2² = 4
  log₂(8)  = 3        because  2³ = 8
  log₂(16) = 4        because  2⁴ = 16
  log₂(32) = 5        because  2⁵ = 32
  log₂(n)  = ?   →   keep halving n until you reach 1; count steps

  log₄(2)  = 0.5      because  4^0.5 = √4 = 2
  log₄(16) = 2        because  4² = 16
  log₄(64) = 3        because  4³ = 64

What is logb(a)? Why does it matter for the Master Theorem? 📄 p. 227

logb(a) is the critical exponent in the Master Theorem. It measures the "effective growth rate due to sub-problems." When we recurse a times into sub-problems of size n/b, the total number of leaves in the recursion tree is alogb(n) = nlogb(a). The comparison between f(n) and nlogb(a) determines which of the three Master Theorem cases applies. 📄 p. 227

Quick computation trick: logb(a) = log(a) / log(b) (using any logarithm base for both). For example, log₄(48) = log₂(48)/log₂(4) = log₂(48)/2 ≈ 5.58/2 ≈ 2.79.

What is a recursion tree? 📄 pp. 227–228

A recursion tree is a visual way to understand a recurrence by unrolling it level by level. The root of the tree represents the top-level call T(n). It has a children (one per sub-problem), each representing T(n/b). Those children each have a children representing T(n/b²), and so on until we reach leaves of size T(1). 📄 pp. 227–228

  Recursion tree for T(n) = 2T(n/2) + n:

  Level 0:              T(n)                   Work: n
                       /    \
  Level 1:         T(n/2)  T(n/2)              Work: n/2 + n/2 = n
                   /  \    /  \
  Level 2:   T(n/4).. T(n/4)..               Work: 4·(n/4) = n
              ...
  Level log n:  1  1  1  1 ... 1              Work: n·1 = n (n leaves)

  Total levels: log n + 1
  Work per level: n
  Total: n · (log n + 1) = O(n log n)
SOURCE: 📄 pp. 79–90, 227–228

What is the "critical exponent" nlogb(a)? 📄 p. 227

The critical exponent nlogb(a) represents the total work done at the leaf level of the recursion tree. At depth logb(n), there are exactly alogb(n) = nlogb(a) leaves (each doing Θ(1) work). This is the cost of the sub-problem decomposition. The Master Theorem compares f(n) (work per level at the root) to nlogb(a) (work at the leaves) to determine which dominates. 📄 p. 227

What is "polynomially larger/smaller"? 📄 p. 227

We say f(n) is polynomially larger than g(n) if f(n) ≥ c·nε·g(n) for some positive constants c and ε. In simpler terms: f(n)/g(n) grows at least as fast as some power nε. A logarithm factor (like log n) is NOT a polynomial factor — this is crucial for the non-applicable case. 📄 p. 227

Example: n² is polynomially larger than n¹ (ratio = n = n¹). But n² log n is NOT polynomially larger than n² (ratio = log n, which grows slower than any nε for ε > 0).

What is Θ (theta) notation? 📄 p. 39

Θ(f(n)) means "grows at exactly the same rate as f(n)" — both an upper bound (O) and lower bound (Ω) simultaneously. When the Master Theorem gives T(n) = Θ(n log n), it means there exist constants c₁, c₂, n₀ such that c₁·n log n ≤ T(n) ≤ c₂·n log n for all n ≥ n₀. 📄 p. 39

What is the substitution method? 📄 p. 79

The substitution method is a way to solve recurrences by: (1) guessing the answer, (2) substituting the guess back into the recurrence, and (3) using induction to verify the constants work out. For example, guess T(n) = O(n log n) for T(n) = 2T(n/2) + n, then substitute T(n/2) ≤ c(n/2)log(n/2) and verify T(n) ≤ cn log n. 📄 p. 79

What is the Akra-Bazzi theorem? 📄 p. 228

The Akra-Bazzi theorem is a generalization of the Master Theorem that handles more complex recurrences where the sub-problem sizes aren't exactly n/b, or where there are multiple terms with different splitting factors. It uses an integral formula: T(n) = Θ(np(1 + ∫₁ⁿ g(u)/up+1 du)), where p satisfies Σaibip = 1. You won't need to apply this formula from scratch, but knowing it exists for irregular recurrences is useful. 📄 p. 228

§1 — What & Why, in Plain English

📘 LECTURE PDF COVERAGE: pp. 79–104 (Lectures 10–11), pp. 227–238 (Lecture 23)

1.1 The fundamental problem — why not just count every operation? 📄 p. 79

Imagine you are a head chef who splits a large pizza-dough ball among multiple junior chefs, each of whom does the same split with their piece, recursively, until each piece is small enough to stretch by hand. How long does it take total? You could count every hand-stretch — but that's tedious. Instead, you ask: how many times do I split, how much work happens at each split level, and which level does the most work? That is exactly what the Master Theorem answers for recursive algorithms. 📄 p. 79

Without a framework like the Master Theorem, analyzing a recursive algorithm means unrolling the recurrence by hand for many levels — messy, error-prone, and slow. The Master Theorem is a one-step lookup table: given a = sub-problems, b = size-reduction factor, and f(n) = non-recursive work, it directly tells you T(n). This is why it appears on the graded Assignment 6 and is a near-certain exam question. 📄 p. 227

1.2 The three cases — which level of the tree dominates? 📄 p. 227

The Master Theorem applies to recurrences of the form T(n) = a·T(n/b) + f(n). Think of the recursion tree as a pyramid. There are multiple levels of work: the root (top) does f(n) work, then the next level does work, and so on down to the leaves. The total runtime depends on which level(s) do the most work:

Case 1: Leaves Dominate (Root is small, bottom is big)
Condition: f(n) = O(nlogba - ε) for some ε > 0 — f(n) is polynomially SMALLER than nlogba Result: T(n) = Θ(nlogba)

Plain English: The work at each level grows as we go deeper. The leaves of the recursion tree accumulate by far the most work. The answer is determined by counting all the leaves. 📄 p. 227

Case 2: Every Level Does Equal Work
Condition: f(n) = Θ(nlogba) — f(n) and nlogba grow at exactly the same rate Result: T(n) = Θ(nlogba · log n) = Θ(f(n) · log n)

Plain English: Every level of the tree contributes the same total work. Since there are log n levels, we multiply f(n) by log n. 📄 p. 227

Case 3: Root Dominates (Root is big, bottom is small)
Condition: f(n) = Ω(nlogba + ε) for some ε > 0 — f(n) is polynomially LARGER than nlogba AND: a·f(n/b) ≤ c·f(n) for some c < 1 (regularity condition) Result: T(n) = Θ(f(n))

Plain English: The work at each level shrinks as we go deeper. The root level does by far the most work, and the total cost is just f(n). 📄 p. 227

Non-applicable: Gap in the Middle
f(n) = nlogba · logkn for k > 0 (log factor bridges the polynomial gap) None of the three cases applies in standard form!

Plain English: f(n) is between Case 2 and Case 3 — it's bigger than nlogba but only by a logarithm factor, which is NOT a polynomial factor. The Master Theorem in its basic form cannot handle this. 📄 p. 227

Extended MT result: For T(n) = aT(n/b) + nlogba·logkn → T(n) = Θ(nlogba·logk+1n). 📄 p. 227

1.3 The recursion tree — a staircase of work 📄 pp. 227–228

Imagine a staircase. Each step is a level of the recursion tree. At the top step, the algorithm does f(n) work. At the next step down, it does a calls of f(n/b) work each — a total of a·f(n/b). At the step below that: a²·f(n/b²). And so on until the bottom step, where nlogba leaves each do Θ(1) work. 📄 pp. 227–228

  Recursion tree for T(n) = a·T(n/b) + n^c:

  Level 0 (root):    [   n^c   ]               Work = n^c
                      /   ...   \
  Level 1:         [n^c/b^c] × a               Work = a × (n/b)^c = a/b^c × n^c
                   /         \
  Level 2:         × a²                        Work = (a/b^c)² × n^c
  ...
  Level j:         × a^j                       Work = (a/b^c)^j × n^c
  ...
  Level log_b(n)   [1] × n^(log_b a)           Work = n^(log_b a) (the leaves)

  Total levels: log_b(n)
  Geometric series ratio per level: r = a / b^c

  If r < 1:  root dominates (geometric series → root = O(f(n)))   → Case 3
  If r = 1:  each level equal (log_b(n) levels × f(n) each)       → Case 2
  If r > 1:  leaves dominate (series grows → leaves = O(n^log_b a)) → Case 1
SOURCE: 📄 pp. 227–228

The ratio r = a/bc tells you whether work grows or shrinks as we descend the tree. r = a/bc > 1 ↔ a > bc ↔ Case 1. r = 1 ↔ a = bc ↔ Case 2. r < 1 ↔ a < bc ↔ Case 3. 📄 p. 227

1.4 Technical definition — the standard Master Theorem form 📄 p. 227

Let a ≥ 1 and b > 1 be constants, let f(n) be a function, and let T(n) be defined on non-negative integers by the recurrence T(n) = a·T(n/b) + f(n), with T(1) = Θ(1). Let n∗ = nlogba be the critical polynomial.

  • Case 1: If f(n) = O(n∗ − ε) for some constant ε > 0, then T(n) = Θ(n∗). 📄 p. 227
  • Case 2: If f(n) = Θ(n∗), then T(n) = Θ(n∗ log n). 📄 p. 227
  • Case 3: If f(n) = Ω(n∗ + ε) for some constant ε > 0, AND a·f(n/b) ≤ c·f(n) for some c < 1 and sufficiently large n, then T(n) = Θ(f(n)). 📄 p. 227
⚠️ Common Point of Confusion — Know This Before §2 The three cases are NOT about comparing a to bc directly (though that's a useful shortcut). They are about comparing f(n) to nlogba ASYMPTOTICALLY with a polynomial gap. A logarithm factor does NOT constitute a polynomial gap — this is why n² log n is NOT polynomially larger than n², making T(n) = 4T(n/2) + n²log n non-applicable. 📄 p. 227

2a Case 1: Leaves Dominate — The Strassen Comparison (A6 Q1) Lecture PDF: pp. 227–238 · Assignment 6 Q1 📋 A6 Q1

Intuition: More chefs at the bottom than the top 📄 p. 227

In Case 1, every time you split the problem, you create MORE total work than you had before. Imagine a=48 chefs, each taking on a quarter of the pizza (b=4). At the next level you have 48²=2304 chefs, each working on a 16th of the pizza. The number of chefs grows exponentially faster than the pizza shrinks — so the bottom level (the leaves, with thousands of tiny tasks) dominates the total work. The root-level work (just one call) is a tiny fraction of the total. 📄 p. 227

Mechanism: How to apply Case 1 📄 p. 227

Step 1: Identify a, b, and f(n).
Read T(n) = a·T(n/b) + f(n). Write down a, b, and what f(n) is.
Step 2: Compute the critical polynomial nlogba.
Calculate logb(a). If a = bc, then logb(a) = c exactly. Otherwise, compute log(a)/log(b).
Step 3: Compare f(n) to nlogba.
For Case 1: check if f(n) = O(nlogba - ε) for some ε > 0. In practice: if f(n) = nc and c < logb(a), then Case 1 applies.
Step 4: State the answer.
T(n) = Θ(nlogba). The answer is the critical polynomial.

Worked Example — A6 Q1: The Strassen Comparison 📄 pp. 227, 239–248 📋 A6 Q1

Problem: Consider T(n) = a·T(n/4) + Θ(n²). Find the largest integer value of a such that this algorithm beats Strassen's Θ(nlog₂7) ≈ Θ(n2.807). 📋 A6 Q1

  SETUP:  T(n) = a·T(n/4) + Θ(n²)
  b = 4,  c = 2,  f(n) = n²

  We need to find: for which values of a is T(n) FASTER than Strassen?

  STEP 1: Compute critical polynomial  n^(log₄ a)
  STEP 2: We need  n^(log₄ a) < n^(log₂ 7) for the complexity to beat Strassen
  STEP 3: This requires  log₄(a) < log₂(7)
  STEP 4: Solve for a:   a < 4^(log₂ 7)

  Computing 4^(log₂ 7):
    4 = 2²,  so  4^(log₂ 7) = (2²)^(log₂ 7) = 2^(2·log₂ 7) = (2^(log₂ 7))² = 7² = 49

  So we need a < 49.
  Largest INTEGER a is  a = 48.  ✓
SOURCE: 📄 pp. 227, 239–248

Now let's verify that a = 48 indeed falls under Case 1 of the Master Theorem:

  a = 48,  b = 4,  f(n) = n²  (so c = 2)

  STEP 1: n^(log₄ 48)
    log₄(48) = log₂(48) / log₂(4) = log₂(48) / 2
    log₂(48): 2^5 = 32, 2^6 = 64, so log₂(48) ≈ 5.585
    log₄(48) ≈ 5.585 / 2 ≈ 2.792

  STEP 2: Compare f(n) = n² to critical exponent n^2.792
    f(n) = n² is polynomially SMALLER than n^2.792
    (ratio: n^2.792 / n² = n^0.792 → ∞ — yes, polynomial gap)
    So f(n) = O(n^(2.792 - ε)) for any ε ∈ (0, 0.792) — e.g., ε = 0.4 ✓

  STEP 3: CASE 1 APPLIES.
    T(n) = Θ(n^(log₄ 48)) ≈ Θ(n^2.792)

  STEP 4: Compare to Strassen Θ(n^log₂ 7) ≈ Θ(n^2.807)
    2.792 < 2.807  →  n^2.792 grows SLOWER than n^2.807  ✓
    Our algorithm with a=48 beats Strassen!

  STEP 5: Check a = 49 (would fail):
    log₄(49) = log₂(49)/2 = log₂(7²)/2 = 2·log₂(7)/2 = log₂(7) ≈ 2.807
    T(n) = Θ(n^(log₂ 7)) — exactly equal to Strassen, does NOT beat it.

  CONCLUSION: Largest a is 48. ✓
SOURCE: 📄 pp. 227, 239–248

⚠️ A6 Q1 Trap: How Did We Get 49? The key insight is that 4^(log₂7) = 7² = 49 — this uses the identity (b^k)^x = b^(kx). Students who try to compute 4^2.807 directly on a calculator get confused. The clean algebraic path: 4 = 2², so 4^(log₂7) = (2²)^(log₂7) = 2^(2log₂7) = (2^(log₂7))² = 7² = 49. 📄 pp. 227, 239–248
2b Case 2: Every Level Contributes Equally (A6 Q2) Lecture PDF: p. 227 · Assignment 6 Q2 📋 A6 Q2

Intuition: A perfectly balanced staircase 📄 p. 227

Case 2 is the "Goldilocks" case — not too many leaves, not too few. The work per level is exactly the same at every step of the staircase. Since there are logb(n) levels in total, the answer is just: (work per level) × (number of levels) = f(n) × log n. 📄 p. 227

This is exactly what happens in merge sort: each level of the recursion tree does exactly n operations total (n/2 from the left merge, n/2 from the right), and there are log₂(n) levels, so the total is n log n. 📄 pp. 79–90

2c Case 3: Root Dominates — T(n) = T(n/2) + n Lecture PDF: p. 227 (Lecture 23)

Intuition: One big chef at the top 📄 p. 227

In Case 3, there is only one recursive call (or very few), and the non-recursive work f(n) is large — much larger than the total work done by all sub-problems combined. Like a single head chef who does most of the cooking while the sous-chef handles a tiny fraction, the root level completely dominates. 📄 p. 227

Worked Example — T(n) = T(n/2) + n 📄 p. 227

  GIVEN:  T(n) = T(n/2) + n
  READ:   a = 1,  b = 2,  f(n) = n

  ────────────────────────────────────────────────────────────────

    log₂(1) = 0  because  2^0 = 1

  ────────────────────────────────────────────────────────────────

    f(n) = n = Ω(1 · n^ε) for ε = 1 — yes, n is polynomially LARGER than 1

  STEP 3: Check regularity condition:
    a·f(n/b) = 1·(n/2) = n/2 ≤ (1/2)·n = c·f(n) with c = 1/2 < 1 ✓

  ────────────────────────────────────────────────────────────────
  STEP 4: CASE 3 APPLIES.
    T(n) = Θ(f(n)) = Θ(n)

  ────────────────────────────────────────────────────────────────

    Level 0: n        (1 node does n work)
    Level 1: n/2      (1 node does n/2 work)

2d Non-applicable Case: The Log Factor Gap (A6 Q3) Lecture PDF: p. 227 · Assignment 6 Q3 📋 A6 Q3

Intuition: A gap too small for the Master Theorem to see 📄 p. 227

The Master Theorem needs a clear "polynomial gap" between f(n) and nlogba. A log factor is NOT a polynomial — it grows much slower than any nε. So when f(n) = n²logn and nlogba = n², the ratio is log n: too big for Case 1 (f > nlogba), too different for Case 2 (not Θ(n²)), but not a polynomial bigger for Case 3. The Master Theorem cannot classify this situation. 📄 p. 227

Worked Example — A6 Q3: T(n) = 4T(n/2) + n²logn 📄 p. 227 📋 A6 Q3

  GIVEN:  T(n) = 4·T(n/2) + n²·log n
  READ:   a = 4,  b = 2,  f(n) = n²·log n

  ────────────────────────────────────────────────────────────────
  STEP 1: Compute critical polynomial n^(log₂ 4)

    log₂(4) = 2  because  2² = 4

    Critical polynomial: n^2 = n²

  ────────────────────────────────────────────────────────────────
  STEP 2: Compare f(n) = n²·log n to critical polynomial n²

    Ratio: f(n) / n² = n²·log n / n² = log n

    Is this ratio at least n^ε for some ε > 0?  NO!
    lim (log n) / n^ε = 0 for any ε > 0  (log grows slower than any polynomial)

    So f(n) = n²·log n is NOT polynomially larger than n².  → Case 3 fails.

  ────────────────────────────────────────────────────────────────
  STEP 3: Check Case 2 — is f(n) = Θ(n²)?

    f(n) = n²·log n,  critical = n²
    n²·log n / n² = log n → ∞   →  n²·log n ≠ Θ(n²)     → Case 2 fails.

  ────────────────────────────────────────────────────────────────
  STEP 4: Check Case 1 — is f(n) = O(n^(2-ε)) for some ε > 0?

    n²·log n ≥ n²  for all n ≥ 1.
    So n²·log n is NOT O(n^(2-ε)) for any ε > 0.           → Case 1 fails.

  ────────────────────────────────────────────────────────────────
  CONCLUSION: ALL THREE CASES FAIL.
  Standard Master Theorem is NOT APPLICABLE.

  Extended MT for this form: T(n) = Θ(n^(log_b a) · log^(k+1) n)
  Here: log_b a = 2, k = 1 (since f(n) = n^2 · log^1 n)
  So: T(n) = Θ(n² · log² n)

📋 A6 Q1 T(n) = aT(n/4) + Θ(n²). Find largest a beating Strassen's Θ(nlog₂7).

▶ Key Points in the Solution
  • Core computation: Need log₄(a) < log₂(7) for our algorithm to grow slower than Strassen. So a < 4log₂7 = (2²)log₂7 = 22log₂7 = (2log₂7)² = 7² = 49. 📄 p. 227
  • Largest integer: a = 48. Then T(n) = Θ(nlog₄48) ≈ Θ(n2.792) < Θ(n2.807). 📄 p. 227
  • Verify a=48 uses Case 1: b=4, b²=16, a=48 > 16 = bc where c=2. So f(n)=n² is polynomially smaller than nlog₄48 ≈ n2.792. Case 1 applies. 📄 p. 227
  • Verify a=49 ties: log₄(49) = log₄(7²) = 2·log₄(7) = 2·log₂(7)/log₂(4) = 2·log₂(7)/2 = log₂(7). So a=49 gives T(n) = Θ(nlog₂7) — exactly equal to Strassen, not better. 📄 pp. 227, 239

▶ Key Points in the Solution

📋 A6 Q3 Explain why the Master Theorem does NOT apply to T(n) = 4T(n/2) + n²logn.

▶ Key Points in the Solution
  • Setup: a=4, b=2, f(n)=n²logn. Critical polynomial: nlog₂4 = n². 📄 p. 227
  • Case 1 fails: Need f(n) = O(n2-ε). But n²logn ≥ n², so f is NOT smaller than n². ✗ 📄 p. 227

Misconception Cards 📄 pp. 227–238

❌ Misconception: "Case 1 applies when a > bc regardless of what f(n) is."

✅ Reality: The shortcut a > bc works ONLY when f(n) is a simple polynomial nc. The real condition is f(n) = O(nlogba - ε). When f(n) has log factors, you must use the proper comparison. 📄 p. 227

❌ Misconception: "Case 2 applies whenever f(n) and nlogba have the same leading exponent."

✅ Reality: Case 2 requires f(n) = Θ(nlogba) — an EXACT asymptotic match including all non-polynomial factors. f(n) = n²logn is NOT Θ(n²) because n²logn/n² = logn → ∞ (not bounded by a constant). 📄 p. 227

❌ Misconception: "For T(n) = 2T(n/4) + √n, the answer is Θ(√n)."

✅ Reality: The answer is Θ(√n · log n). Case 2 always introduces a multiplicative log n factor. This is the most common single-point loss on the exam. 📄 p. 227

❌ Misconception: "The Master Theorem applies to ALL recurrences of the form T(n) = aT(n/b) + f(n)."

✅ Reality: The MT has a gap: when f(n) = nlogba·poly-log(n) (log factors separate f from the critical polynomial), none of the three cases applies. The extended MT or Akra-Bazzi must be used. 📄 p. 227

❌ Misconception: "Binary search uses Case 3 of the Master Theorem."

✅ Reality: Binary search T(n)=T(n/2)+1 uses Case 2 (equal). nlog₂1=1=f(n)=1. Equal → Case 2 → Θ(1·logn) = Θ(logn). This is a common confusion because a=1 feels like "nothing recurses." 📄 pp. 55–62

❌ Misconception: "log₄(48) is just 48/4 = 12."

✅ Reality: log₄(48) asks "4 to what power gives 48?" NOT "48 divided by 4." log₄(48) = log₂(48)/log₂(4) ≈ 5.585/2 ≈ 2.79. Division and logarithm are completely different operations. 📄 p. 227

Memory Aids & Mnemonics 📄 pp. 227–238

🧠 Mnemonic 1: "BIG BOTTOM, BIG MIDDLE, BIG TOP" Case 1 = BIG BOTTOM (leaves dominate) = answer is nlogba (the leaf count)
Case 2 = BIG MIDDLE (everyone equal) = answer is nlogba·log n (multiply by levels)
Case 3 = BIG TOP (root dominates) = answer is f(n) (the root work)
The dominant level's work = the answer. 📄 p. 227
🧠 Mnemonic 2: "A vs B-to-the-C" When f(n) = nc exactly: compare a to bc.
a > bc → Case 1 (subproblems overwhelm combine work)
a = bc → Case 2 (balance)
a < bc → Case 3 (combine work overwhelms subproblems)
Shortcut: "a vs b-to-the-c" — three comparisons, three cases. 📄 p. 227
🧠 Mnemonic 3: "CASE 2 + 1 LOG" The only tricky case to remember is Case 2. The formula is ALWAYS (critical polynomial) × log n. Write "+1 LOG" next to your Case 2 formula on the cheat sheet. Never forget the log n factor. 📄 p. 227
🧠 Mnemonic 4: "Log-Not-Poly Gap = Non-Applicable" If f(n)/nlogba = some power of log n (like log n, log²n, log³n), the standard MT cannot apply. Log is "not a polynomial gap." Use extended MT: just increment the log exponent by 1. n²logn → Θ(n²log²n). n²log²n → Θ(n²log³n). 📄 p. 227
🧠 Mnemonic 5: "7 squared is 49" (Strassen magic number) For the A6 Q1 class of problem: 4log₂7 = 7² = 49. Memorize this identity. The key algebraic move: 4 = 2², so 4log₂7 = (2²)log₂7 = 22log₂7 = (2log₂7)² = 7² = 49. Largest a that beats Strassen (with b=4) is 48. 📄 pp. 227, 239

Q11. Write pseudocode for merge sort on an array A[0..n-1]. Then (a) identify the values of a, b, and f(n), (b) apply the Master Theorem, and (c) verify your answer matches the known O(n log n) bound.

▶ Show Answer 
function MergeSort(A, left, right):
    # Sort A[left..right-1] in place
    # Base case: 0 or 1 elements — already sorted
    if right - left <= 1:
        return

    # Divide: find midpoint
    mid = (left + right) // 2

    # Conquer: recursively sort each half     ← contributes 2·T(n/2)
    MergeSort(A, left, mid)
    MergeSort(A, mid, right)

    # Combine: merge the two sorted halves   ← contributes O(n) work
    Merge(A, left, mid, right)

function Merge(A, left, mid, right):
    # Standard two-pointer merge into temp array; O(n) total operations
    L = A[left..mid-1];  R = A[mid..right-1]
    i = 0; j = 0; k = left
    while i < len(L) and j < len(R):
        if L[i] <= R[j]:  A[k] = L[i]; i += 1
        else:              A[k] = R[j]; j += 1
        k += 1
    copy remaining L or R into A[k..]

function ApplyMasterTheorem(a, b, c):
    # Given: T(n) = a*T(n/b) + n^c
    # Computes: the Θ answer using Master Theorem

    # Step 1: Compute the critical exponent
    log_b_a = log(a) / log(b)          # log_b(a) using change of base

    # Step 2: Compare c (exponent of f) to log_b_a (exponent of critical)
    epsilon = 0.001                     # practical threshold for "polynomial gap"

    if c < log_b_a - epsilon:           # Case 1: f is polynomially smaller
        print("Case 1 applies")
        print("T(n) = Theta(n^" + log_b_a + ")")

    elif abs(c - log_b_a) <= epsilon:   # Case 2: f and critical match
        print("Case 2 applies")
        print("T(n) = Theta(n^" + c + " * log n)")

    else:                               # Case 3: f is polynomially larger
        # Check regularity: a * f(n/b) <= d * f(n) for d < 1
        # For f(n) = n^c: regularity holds iff a * (1/b)^c < 1, i.e., a < b^c
        regularity_ratio = a / (b ** c)
        if regularity_ratio < 1:        # a < b^c ensures regularity
            print("Case 3 applies")
            print("T(n) = Theta(n^" + c + ")")
        else:
            print("WARNING: verify regularity condition manually")

  3. For Case 3, verify regularity: for f(n)=nc, regularity holds iff a < bc (equivalently logb(a) < c — same as the Case 3 condition). So for polynomial f, the regularity condition is automatic. 📄 p. 227

You are given a choice between three divide-and-conquer algorithms for computing the product of two n-bit integers: Algorithm A: T(n) = 4T(n/2) + O(n) Algorithm B: T(n) = 3T(n/2) + O(n) (Karatsuba) Algorithm C: T(n) = 2T(n/2) + O(n²) (a) Apply the Master Theorem to each. (b) Order them from fastest to slowest. (c) Prove that Algorithm B is faster than Algorithm A using the definition of asymptotic ordering. (d) For input sizes n=1000 and n=1,000,000, estimate the practical advantage of B over A.

▶ Show Answer

(a) Master Theorem for each: 📄 p. 227

A: 4T(n/2)+n42nn²Case 1Θ(n²)
B: 3T(n/2)+n (Karatsuba)32nnlog₂3≈n1.585Case 1Θ(nlog₂3)

(b) Order fastest to slowest:

log₂(3) ≈ 1.585. So rankings: n1.585 < n² = n². Algorithm B (Karatsuba) > Algorithms A and C (tied at n²). Ordering: B fastest, then A and C equally slow. 📄 pp. 227–238

(c) Proof that B is faster than A:

T_A(n) = Θ(n²) and T_B(n) = Θ(nlog₂3). We need to show nlog₂3 = o(n²) (B is asymptotically faster). 📄 pp. 39–42

Compute lim T_B(n)/T_A(n) = lim nlog₂3/n² = lim nlog₂3 - 2 = lim n-(2-log₂3).

Since log₂3 ≈ 1.585 < 2, we have 2 - log₂3 ≈ 0.415 > 0. So n-(0.415) → 0 as n → ∞.

Therefore T_B(n) = o(T_A(n)) — Algorithm B is strictly faster. ✓ 📄 pp. 39–42

(d) Practical advantage at n=1000 and n=1,000,000:

  At n = 1,000:
    A: Θ(n²) ≈ 10⁶ operations
    B: Θ(n^1.585) ≈ 1000^1.585 = 10^(3×1.585) = 10^4.755 ≈ 57,000 operations
    Ratio: A/B ≈ 10⁶/57000 ≈ 17.5× advantage for B

  At n = 1,000,000:
    A: Θ(n²) ≈ 10¹²
    B: Θ(n^1.585) ≈ (10⁶)^1.585 = 10^(6×1.585) = 10^9.51 ≈ 3.2×10⁹
    Ratio: A/B ≈ 10¹² / 3.2×10⁹ ≈ 312× advantage for B

  Conclusion: The advantage grows with input size. At n=10⁶, Karatsuba
  is over 300× faster — this explains why it's used in cryptographic
  libraries for large-integer arithmetic.

§5 — Topic Cheat Sheet

📘 SOURCE: Lecture 23, pp. 227–238 · Lectures 10–11, pp. 79–104 · A6 Q1–Q3

IN ONE SENTENCE, THIS TOPIC IS:

The Master Theorem is a formula that solves recurrences T(n) = aT(n/b) + f(n) by comparing f(n) to the critical polynomial nlogba to determine which level of the recursion tree dominates the total work. 📄 p. 227

THE THREE THINGS YOU MUST KNOW FOR THE EXAM:

  1. Compute nlogba first. Always start here. Then compare f(n) to it polynomially. Never try to apply cases without this computation. 📄 p. 227
  2. Case 2 adds log n. Always. If f(n) = Θ(critical), then T(n) = Θ(critical · log n). The log factor is mandatory. Writing Θ(√n) for T=2T(n/4)+√n costs points. 📄 p. 227
  3. Log-gap → non-applicable. Explain all three failures. If f(n)/nlogba = logn (or any log power), show each case fails in turn. Then optionally apply extended MT: increment the log exponent by 1. 📄 p. 227

THE CANONICAL ALGORITHM (Master Theorem Application):

To solve T(n) = a·T(n/b) + f(n):

1. Compute  n* = n^(log_b(a))  using: log_b(a) = log(a)/log(b)
2. Compare f(n) to n*:
   a > b^c (f=n^c smaller):  Case 1 → T(n) = Theta(n*)
   a = b^c (f=n^c equal):    Case 2 → T(n) = Theta(n* · log n)
   a < b^c (f=n^c larger):   Case 3 → T(n) = Theta(f(n))
   f = n* · log^k n:          NON-APPLICABLE → extended MT: Theta(n* · log^(k+1) n)

Key identity for Strassen problems: 4^(log_2(7)) = 7^2 = 49
Largest a beating Strassen with b=4: a = 48

THE #1 MISTAKE ON EXAM QUESTIONS ABOUT THIS TOPIC:

Writing Θ(nc) instead of Θ(nc·log n) for Case 2 problems. When you correctly identify Case 2 (f(n) = Θ(nlogba)), the answer ALWAYS has an additional log n factor. This applies to merge sort (Θ(n log n), NOT Θ(n)) and to A6 Q2 (Θ(√n · log n), NOT Θ(√n)). The log n factor represents the number of equally-weighted levels in the recursion tree. 📄 p. 227

IF YOU REMEMBER NOTHING ELSE, REMEMBER THIS:

Compute nlogba. Compare f(n) to it polynomially. f smaller → Case 1 (leaves win). f equal → Case 2 (add log n). f larger → Case 3 (root wins). f = critical × logkn → Non-applicable (use extended MT, add one more log). 📄 p. 227
