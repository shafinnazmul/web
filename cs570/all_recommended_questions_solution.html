KT 1.1 · Q1
Understanding Unstable Pairs — Definition & Verification
Problem: Given the following preference lists, determine whether the matching M = {(S₀,C₁), (S₁,C₀), (S₂,C₂)} is stable. If it is not, find an unstable pair.

Advisor preferences (left = most preferred): C₀: S₁ > S₀ > S₂  |  C₁: S₀ > S₁ > S₂  |  C₂: S₀ > S₁ > S₂
Student preferences: S₀: C₁ > C₀ > C₂  |  S₁: C₀ > C₁ > C₂  |  S₂: C₀ > C₁ > C₂
▶ Show Full Answer & Explanation

What is a stable matching? A matching is stable if there is NO "unstable pair" — two participants who would both prefer each other over their current partners. An unstable pair (Sᵢ, Cⱼ) requires:

  1. Sᵢ prefers Cⱼ over Sᵢ's current advisor.
  2. Cⱼ prefers Sᵢ over Cⱼ's current student.

Step 1: Write out who is matched with whom

  M = { S₀ ↔ C₁,  S₁ ↔ C₀,  S₂ ↔ C₂ }

  C₀ has: S₁
  C₁ has: S₀
  C₂ has: S₂

Step 2: Check every possible pair (Sᵢ, Cⱼ) where Sᵢ is NOT matched with Cⱼ

Pair to testDoes Sᵢ prefer Cⱼ over current?Does Cⱼ prefer Sᵢ over current?Unstable?
(S₀, C₀)S₀ has C₁. S₀'s list: C₁>C₀>C₂. C₀ is ranked 2nd, C₁ is 1st. S₀ does NOT prefer C₀.—❌ No
(S₀, C₂)S₀'s list: C₁>C₀>C₂. C₂ is ranked 3rd, C₁ is 1st. S₀ does NOT prefer C₂.—❌ No
(S₁, C₁)S₁ has C₀. S₁'s list: C₀>C₁>C₂. C₀ is ranked 1st. S₁ does NOT prefer C₁.—❌ No
(S₂, C₀)S₂ has C₂. S₂'s list: C₀>C₁>C₂. C₀ is ranked 1st, C₂ is 3rd. S₂ DOES prefer C₀!C₀ has S₁. C₀'s list: S₁>S₀>S₂. S₁ is 1st, S₂ is 3rd. C₀ does NOT prefer S₂.❌ No
(S₂, C₁)S₂ prefers C₁ (rank 2) over C₂ (rank 3)? Yes!C₁ has S₀. C₁'s list: S₀>S₁>S₂. S₀ is 1st, S₂ is 3rd. C₁ does NOT prefer S₂.❌ No
✅ Result No unstable pair was found. The matching M = {(S₀,C₁), (S₁,C₀), (S₂,C₂)} is stable. Every potential "rogue couple" is blocked because at least one party prefers their current partner. 📄 p. 19

Key Insight for the Exam

To check stability: for each pair (Sᵢ,Cⱼ) not in the matching, check both conditions. If even ONE condition fails, the pair cannot defect — they are blocked. All pairs must be blocked for the matching to be stable.

KT 1.1 · Q2
Gale-Shapley Algorithm Trace
Problem: Run the Gale-Shapley algorithm (student-proposing) on the following instance. Show every proposal made, every acceptance/rejection, and the final stable matching.

S₀: C₀>C₁>C₂  |  S₁: C₁>C₀>C₂  |  S₂: C₀>C₁>C₂
C₀: S₁>S₀>S₂  |  C₁: S₀>S₁>S₂  |  C₂: S₀>S₁>S₂
▶ Show Full Answer & Explanation

How Gale-Shapley works (student-proposing version):

  1. Each unmatched student proposes to their most-preferred advisor they haven't yet tried.
  2. Each advisor tentatively holds their best proposal so far and rejects the rest.
  3. Rejected students move to their next choice on the list.
  4. Repeat until all students are matched.
  ROUND 1:
    S₀ proposes to C₀ (top of S₀'s list)
    S₁ proposes to C₁ (top of S₁'s list)
    S₂ proposes to C₀ (top of S₂'s list)

    C₀ receives proposals from S₀ and S₂.
      C₀'s preference: S₁ > S₀ > S₂.
      Between S₀ (rank 2) and S₂ (rank 3): C₀ prefers S₀.
      C₀ tentatively holds S₀, rejects S₂.

    C₁ receives proposal from S₁.
      C₁ tentatively holds S₁.

  After Round 1: C₀⟵S₀, C₁⟵S₁, S₂ is unmatched and rejected.

  ROUND 2:
    S₂ was rejected by C₀. Next on S₂'s list: C₁.
    S₂ proposes to C₁.

    C₁ now holds S₁ (rank 2 on C₁'s list).
    New proposal from S₂ (rank 3 on C₁'s list).
    C₁ prefers S₁ (rank 2) over S₂ (rank 3). C₁ rejects S₂.

  After Round 2: C₀⟵S₀, C₁⟵S₁, S₂ still unmatched.

  ROUND 3:
    S₂ was rejected by C₁. Next on S₂'s list: C₂.
    S₂ proposes to C₂.

    C₂ was empty. C₂ tentatively holds S₂.

  After Round 3: All students matched!
  FINAL MATCHING: { S₀↔C₀, S₁↔C₁, S₂↔C₂ }
SOURCE: 📄 pp. 19–22
📌 Why this is student-optimal In student-proposing Gale-Shapley, each student ends up with the best advisor they can possibly receive in ANY stable matching. S₀ gets C₀ (top choice), S₁ gets C₁ (top choice). S₂ gets C₂ (last choice) — but S₂ cannot do better: S₂ is last on every advisor's list, so S₂ would be rejected everywhere else first. 📄 p. 22

Complexity of Gale-Shapley

With n students and n advisors: at most n² proposals total (each student proposes to each advisor at most once). So runtime = O(n²). 📄 p. 22

What Makes an Algorithm "Efficient"? — The Polynomial Time Criterion
Problem: Suppose algorithm A₁ solves instances of size n in time 60n·log₂n, and algorithm A₂ solves the same problem in time n². For which values of n is A₁ faster? What does this tell us about the choice of algorithms in practice?
▶ Show Full Answer & Explanation

Step 1: When is A₁ faster?

A₁ is faster when 60n·log₂n < n². Divide both sides by n (n > 0):

60·log₂n < n  ⟺  n > 60·log₂n

For small n this fails (e.g., n=2: 60·1=60 > 2). We need n large enough. Testing:

n60·log₂nnA₁ faster?
2602❌ No
51260×9=540512❌ No
102460×10=6001024✅ Yes (1024>600)
~700+~600700✅ Yes (crossover ≈700)
📌 Key Lesson For n ≳ 700, A₁ (O(n log n)) is faster. For very large inputs (n=10⁶), A₁ runs in ~2×10⁷ steps while A₂ runs in 10¹² steps — A₁ is 50,000 times faster. Asymptotic complexity matters enormously at scale, even when constant factors look large. 📄 pp. 23–30

The Polynomial Time Standard

An algorithm is called efficient if its worst-case running time is polynomial: O(n^k) for some constant k. Why? Because:

  • Scale well: doubling input size increases runtime by a bounded factor.
  • Composable: polynomial × polynomial = polynomial (algorithms can be combined).
  • Machine-independent: the polynomial class is robust across different computational models.

This is the Cobham-Edmonds thesis, referenced in Lecture 25. 📄 p. 249

KT 2.2 · Q1
Formal Proof of a Tight Bound (Θ)
Problem: Prove formally that f(n) = n(n−1)/2 is Θ(n²). Your proof must exhibit explicit constants c₁, c₂, and n₀.
▶ Show Full Answer & Explanation

Recall the Θ definition: f(n) = Θ(g(n)) means there exist positive constants c₁, c₂, n₀ such that c₁·g(n) ≤ f(n) ≤ c₂·g(n) for all n ≥ n₀. Here g(n) = n².

PROOF: n(n−1)/2 = Θ(n²)
Upper bound (O):
n(n−1)/2 = n²/2 − n/2 ≤ n²/2   for all n ≥ 0.
So c₂ = 1/2 works: n(n−1)/2 ≤ (1/2)·n². ✓
Lower bound (Ω):
n(n−1)/2 = n²/2 − n/2.
We need n²/2 − n/2 ≥ c₁·n² for some c₁ > 0 and large n.
n²/2 − n/2 = n²/4 + (n²/4 − n/2) = n²/4 + n(n−2)/4.
For n ≥ 2: n(n−2)/4 ≥ 0, so n(n−1)/2 ≥ n²/4.
So c₁ = 1/4 and n₀ = 2 work. ✓
Conclusion:
For all n ≥ n₀ = 2:   (1/4)n² ≤ n(n−1)/2 ≤ (1/2)n².
Therefore f(n) = n(n−1)/2 = Θ(n²). ∎
SOURCE: 📄 p. 39 (tight bounds example)
💡 Exam Tip Always exhibit c₁, c₂, n₀ explicitly. Saying "the n²/2 term dominates" is informal — the grader wants the inequalities. Also note: for Θ you need BOTH an O upper bound AND an Ω lower bound.
KT 2.2 · Q2
True/False Asymptotic Claims — Prove or Find Counterexample
Problem (from Lecture 6 / Assignment 2 style): For each claim, either prove it TRUE or find a COUNTEREXAMPLE.
(a) If f(n) = O(g(n)) and f(n) = Ω(h(n)), then g(n) = Ω(h(n)).
(b) If f(n) = O(g(n)), then f(n)² = O(g(n)²).
(c) If f(n) = Θ(g(n)), then log f(n) = Θ(log g(n)).
▶ Show Full Answer & Explanation

(a) TRUE — Proof

Given: f = O(g) means f(n) ≤ c_g · g(n) for n ≥ n_g; f = Ω(h) means f(n) ≥ c_h · h(n) for n ≥ n_h.
Chain: For n ≥ max(n_g, n_h): c_h·h(n) ≤ f(n) ≤ c_g·g(n). Therefore c_h·h(n) ≤ c_g·g(n), which gives h(n) ≤ (c_g/c_h)·g(n). So g(n) = Ω(h(n)) with constant c_h/c_g. ✓ ∎

(b) TRUE — Proof

Given: f(n) ≤ c·g(n) for n ≥ n₀.
Square both sides: f(n)² ≤ c²·g(n)² for n ≥ n₀. (Squaring preserves inequality since both sides are non-negative.) So c₀ = c² and n₀ are the witnesses. f(n)² = O(g(n)²). ✓ ∎

(c) FALSE — Counterexample

⚠️ Common Exam Trap This looks true but isn't! The key: Θ is about multiplicative ratios, but log "compresses" differences.
Counterexample: Let f(n) = 2 and g(n) = 1 + 1/n.
f = Θ(g)? Yes: 1 ≤ g(n) ≤ 2, and f(n) = 2, so f = Θ(g) (both are Θ(1)). ✓
log f(n) = log 2 = 1 (a positive constant).
log g(n) = log(1 + 1/n) → 0 as n → ∞.
Ratio: log f(n) / log g(n) = 1 / log(1+1/n) → +∞. So log f ≠ Θ(log g). ✗ ∎

Why it fails: Θ requires the ratio f/g to be bounded between two positive constants. But if f and g are both close to constants (say both ≈ 1), their logs can diverge wildly. 📄 pp. 43–45

Ordering Functions by Growth Rate
Problem (Assignment 4 Q1 style): Order the following functions from slowest-growing to fastest-growing, and identify any that are Θ-equivalent to each other.
g₁(n) = 2^√(log n)  |  g₂(n) = n(log n)²  |  g₃(n) = n^(4/3)  |  g₄(n) = 2^n  |  g₅(n) = 2^(n²)  |  g₆(n) = 2^(2^n)
▶ Show Full Answer & Explanation

Method: Use the limit comparison method. For any two functions f, g: compute lim(n→∞) f(n)/g(n). If limit is 0: f = o(g) (f grows slower). If limit is ∞: f = ω(g). If limit is c∈(0,∞): f = Θ(g).

The Hierarchy (slowest → fastest)

  g₁(n) = 2^√(log n)  ≺  g₂(n) = n(log n)²  ≺  g₃(n) = n^(4/3)  ≺  g₄(n) = 2^n  ≺  g₅(n) = 2^(n²)  ≺  g₆(n) = 2^(2^n)

  Standard ranking: sub-linear poly < poly·log < poly < exponential in n < exponential in n² < doubly exponential

KT 2.4 · Q2
Analyzing Iterative Programs — Complexity from Code
Problem (Lecture 6 examples): For each code fragment, state the exact time complexity and explain your reasoning.
(a) The no-duplicates checker. (b) Matrix multiplication. (c) Number of bits in n.
▶ Show Full Answer & Explanation

(a) No-Duplicates Checker — O(n²)

function noDuplicates(A):
    for i = 0 to n-1:             # outer loop: n iterations
        for j = i+1 to n-1:       # inner loop: at most n-i-1 iterations
            if A[i] == A[j]:
                return False
    return True

Total comparisons = (n-1) + (n-2) + ... + 1 = n(n-1)/2 = Θ(n²). 📄 p. 47

(b) Matrix Multiplication — O(n³)

function matrixMultiply(A, B):
    for i = 0 to n-1:      # n iterations
        for j = 0 to n-1:  # n iterations
            C[i][j] = 0
            for k = 0 to n-1:   # n iterations
                C[i][j] += A[i][k] * B[k][j]
    return C

Three nested loops each running n times: n × n × n = Θ(n³). 📄 p. 49

(c) Number of Bits in n — O(log n)

function numberOfBits(n):
    count = 1
    while n > 1:
        count = count + 1
        n = n div 2     # integer division by 2
    return count

Each iteration halves n. Starting from n, after k iterations we have n/2^k. The loop stops when n/2^k ≤ 1, i.e., k = ⌊log₂n⌋. So the loop runs ⌊log₂n⌋ times: Θ(log n). 📄 p. 51

💡 Pattern to Recognize Halving loops (÷2 each step) → O(log n).
Single nested loops (0..n inside 0..n) → O(n²).
Triple nested loops → O(n³).
Single scan → O(n).

Formal Definition Exercises — O, Ω, Θ, o, ω
Problem (CLRS §3.1 exercises): Is 2^(n+1) = O(2^n)? Is 2^(2n) = O(2^n)? Justify each answer with the formal definition.
▶ Show Full Answer & Explanation

Part (a): Is 2^(n+1) = O(2^n)?

2^(n+1) = 2 · 2^n.
So 2^(n+1) ≤ 2 · 2^n for all n ≥ 0.
This matches the O definition with c = 2, n₀ = 0. ✓
Answer: YES, 2^(n+1) = O(2^n).

Part (b): Is 2^(2n) = O(2^n)?

2^(2n) = (2^n)². This grows as the SQUARE of 2^n.
For 2^(2n) ≤ c · 2^n we'd need c ≥ 2^n for all n — but c must be a FIXED constant.
Formally: lim (2^(2n))/(2^n) = lim 2^n = +∞. So 2^(2n) is not bounded by any constant multiple of 2^n.
Answer: NO, 2^(2n) ≠ O(2^n). In fact 2^(2n) = ω(2^n).
⚠️ Common Mistake Students often think "both are exponentials so they must be comparable as O". Wrong! The BASE raised to a multiple of n grows much faster: 2^(2n) = 4^n. 4^n grows strictly faster than 2^n by a factor of (4/2)^n = 2^n which → ∞.

BFS Shortest-Path Tree — Reconstruction and Properties
Problem: Prove that in an undirected graph G, BFS(s) produces a tree T such that, for every vertex v, the path from s to v in T is a shortest path in G (minimizing number of edges). Your proof must use the level[] array definition.
▶ Show Full Answer & Explanation

Key definitions: BFS maintains level[v] = the number of edges on the path from s to v discovered by BFS. We claim level[v] equals the true shortest-path distance d(s,v) for all reachable v.

PROOF by strong induction on level[v]
Base Case (level = 0): Only s has level[s] = 0. d(s,s) = 0. ✓
Inductive Hypothesis: For all vertices u with level[u] < k, we have level[u] = d(s,u).
Inductive Step: Let v be a vertex with level[v] = k, discovered via neighbor u (level[u] = k−1). By IH: d(s,u) = k−1. The path s→...→u→v has length k. So d(s,v) ≤ k.
Lower bound: Any path from s to v must at some point cross from vertices at distance ≤ k−1 to vertices at distance k. The last edge of this crossing goes from a vertex at distance ≤ k−1 to v, giving distance ≥ k. So d(s,v) ≥ k.
Conclusion: d(s,v) = k = level[v]. ✓ ∎
SOURCE: 📄 pp. 118–128
📌 Why BFS Works (Intuition) BFS explores vertices in non-decreasing order of distance. Level-0 first, then level-1, then level-2... Because we use a FIFO queue: the first time we reach v, we reach it by the shortest possible path. Any later path to v through a longer route would require passing through at least one more un-explored (farther) vertex first.

KT 3.5 · Q1
Topological Sort — Correctness Proof
Problem: Prove that every directed acyclic graph (DAG) has a topological ordering. Your proof must construct the ordering explicitly.
▶ Show Full Answer & Explanation

Lemma needed: Every DAG has at least one vertex with in-degree 0 (no incoming edges).

PROOF of Lemma
Suppose for contradiction every vertex has in-degree ≥ 1. Start at any vertex v₀. Since in-degree(v₀)≥1, there is some vertex v₁ with edge (v₁,v₀). Since in-degree(v₁)≥1, there is some v₂ with edge (v₂,v₁). Continuing, we build an infinite sequence v₀,v₁,v₂,... Each v_i must appear earlier in the sequence (finite graph!), so there is a repeated vertex — a cycle. But G is a DAG. Contradiction. ✓ ∎
PROOF: Every DAG has a topological ordering (by induction on |V|)
Base case n=1: Single vertex. Trivial ordering. ✓
Inductive step: Given DAG G with n vertices. By the Lemma, G has a vertex v with in-degree 0. Place v first in the ordering.
Remove v and all edges from v. The remaining graph G' is still a DAG (removing a vertex cannot create cycles) with n−1 vertices.
By the inductive hypothesis, G' has a topological ordering. Prepend v to get an ordering for G.
Correctness: v has no incoming edges, so no vertex needs to come before v. Every edge of G either leaves v (satisfied since v is first) or is entirely in G' (satisfied by G''s ordering). ✓ ∎
SOURCE: 📄 pp. 141–156
📌 Algorithm from the Proof The proof directly gives us the O(n²) topological sort algorithm: repeatedly find and remove a source (in-degree 0 vertex). With adjacency lists and a queue, this runs in O(n+m). 📄 p. 153
KT 3.5 · Q2
Longest Path in a DAG — Course Prerequisite Problem
Problem: Given the following DAG where each edge (A→B) means "A is a prerequisite for B", find the minimum number of semesters to complete all courses if unlimited courses can be taken per semester.
Edges: (Alg→DP), (Alg→Graphs), (Linear→Alg), (Calc→Linear), (DP→NP), (Graphs→NP).
▶ Show Full Answer & Explanation

Key insight: The minimum number of semesters = the length of the LONGEST PATH in the DAG + 1 (since courses on the same path must be sequential). We compute this using topological order. 📄 pp. 157–168

  GRAPH:

The Construction (Triangle Gadget)

Given a 3-SAT formula with m clauses:

  1. For each clause Cⱼ = (ℓⱼ₁ ∨ ℓⱼ₂ ∨ ℓⱼ₃), create 3 vertices vⱼ₁, vⱼ₂, vⱼ₃.

  3. For complementary literals: if vⱼᵢ = ℓ and vⱼ'ᵢ' = ¬ℓ, add edge (vⱼᵢ, vⱼ'ᵢ').
  4. Set k = m (number of clauses).

Our Formula: Φ = (x₁ ∨ ¬x₂ ∨ x₃) ∧ (¬x₁ ∨ x₂ ∨ ¬x₃)

  Clause C₁ = (x₁ ∨ ¬x₂ ∨ x₃):  vertices v₁₁=x₁, v₁₂=¬x₂, v₁₃=x₃
  Clause C₂ = (¬x₁ ∨ x₂ ∨ ¬x₃): vertices v₂₁=¬x₁, v₂₂=x₂, v₂₃=¬x₃

  Triangle edges (within each clause):
    C₁: (v₁₁,v₁₂), (v₁₁,v₁₃), (v₁₂,v₁₃)
    C₂: (v₂₁,v₂₂), (v₂₁,v₂₃), (v₂₂,v₂₃)

  Conflict edges (complementary literals, between clauses):
    x₁ (v₁₁) ——— ¬x₁ (v₂₁)
    ¬x₂ (v₁₂) ——— x₂ (v₂₂)
    x₃ (v₁₃) ——— ¬x₃ (v₂₃)

Finding IS of size k=2

Try assignment: x₁=TRUE, x₂=FALSE, x₃=TRUE.

  • C₁: x₁=T → TRUE. Pick v₁₁ (represents x₁).
  • C₂: ¬x₁=F, x₂=F, ¬x₃=F → none of these are true! Try x₂: x₂=FALSE, so ¬x₃=FALSE... Hmm. Let's try x₁=FALSE, x₂=TRUE, x₃=TRUE.
  • C₁: x₃=T → TRUE. Pick v₁₃ (represents x₃).
  • C₂: x₂=T → TRUE. Pick v₂₂ (represents x₂).

IS = {v₁₃, v₂₂}. Check: edge between them? v₁₃=x₃, v₂₂=x₂. No conflict edge (not complementary). No triangle edge (different clauses). ✓ IS of size 2! 📄 p. 275

✅ Why the Reduction is Valid The triangle ensures we can pick at most one vertex per clause (any two in the same clause are connected). Conflict edges ensure we don't pick a literal and its negation simultaneously (which would be a contradiction). Picking k=m vertices = one per clause = a satisfying assignment.

Dijkstra's Greedy Choice — Why It Is Correct
Problem: Dijkstra's algorithm makes a greedy choice: at each step it settles the vertex with minimum Distance. Prove this greedy choice is correct for non-negative edge weights. Why does it fail with negative edges?
▶ Show Full Answer & Explanation
PROOF: Dijkstra's greedy choice is optimal (non-negative weights)
Claim: When vertex u is selected (minimum Distance among unsettled), Distance[u] equals the true shortest distance d(s,u).
Proof by contradiction: Suppose Distance[u] > d(s,u). Let P be a true shortest path from s to u.
P must at some point leave the settled set S. Let x be the first unsettled vertex on P, with predecessor y ∈ S.
When y was settled: edge (y,x) was relaxed → Distance[x] ≤ Distance[y] + w(y,x) = cost(s→y→x on P).
The remaining path from x to u has non-negative cost (all weights ≥ 0). So: Distance[x] ≤ cost(s→x on P) ≤ cost(P) = d(s,u).
But u was chosen as minimum among unsettled: Distance[u] ≤ Distance[x] ≤ d(s,u). Contradicts Distance[u] > d(s,u). ✗ ∎

Why Negative Weights Break This

The proof relies on: "remaining path from x to u has non-negative cost." With negative weights, there could be a later edge (z,u) with negative weight such that going through z gives a SHORTER path even though z was discovered later. The greedy commitment becomes invalid.

  Counterexample: S→A (weight 1), S→B (weight 3), A→B (weight -3), B→T (weight 1)

  Dijkstra settles A first (dist=1), then B (dist=min(3, 1-3)=min(3,-2)=-2).
  But wait: Dijkstra commits S-B at distance 3 BEFORE processing A,
  then can't update it when it discovers 1+(-3)=-2 is shorter.
  With a proper relaxation check this would work... actually let's clarify:

  If B is committed (settled) before discovering the path through A,
  its distance gets frozen incorrectly. The greedy "settle minimum" rule
  commits B=3 before realizing A→B gives B=-2. Negative weight = problem. ✓

MST Property — Minimum Connected Spanning Subgraph = Tree
Problem (Assignment 6 Q6): Prove: In a connected undirected graph G where all edge weights are strictly positive, any subset of edges that (a) connects all n vertices and (b) has minimum total weight must be a spanning tree (exactly n−1 edges, no cycles).
▶ Show Full Answer & Explanation
PROOF: Minimum Connected Spanning Subgraph with Positive Weights = Tree
Step 1: Let T be any connected spanning subgraph with minimum total edge weight.
Step 2 (No cycles): Suppose T contains a cycle C = (v₁→v₂→...→v_k→v₁). Pick any edge e = (vᵢ, vᵢ₊₁) ∈ C.
Step 3: Let T' = T \ {e} (remove e from T). Claim T' is still connected: any path that used edge e can reroute through the rest of the cycle (go vᵢ → vᵢ₋₁ → ... → v₁ → v_k → ... → vᵢ₊₁).
Step 4: weight(T') = weight(T) − weight(e) < weight(T) since weight(e) > 0 (all weights positive).
Step 5 (Contradiction): T' is a connected spanning subgraph with strictly less weight than T. But T was assumed minimum. Contradiction. ✗
Conclusion: T has no cycles. A connected acyclic graph on n vertices has exactly n−1 edges = a spanning tree. ✓ ∎
SOURCE: 📄 p. 227
⚠️ Requires Positive Weights! If edge weights can be zero, removing a zero-weight cycle edge doesn't reduce total weight — the contradiction fails. The result is false for non-positive weights: a graph could have a minimum subgraph that is a cycle (all cycle edges have weight 0).

Bellman-Ford — Why n−1 Iterations Suffice
Problem: Prove that after k iterations of Bellman-Ford, Distance[v] ≤ δₖ(s,v) for all v, where δₖ(s,v) is the true shortest-path cost using at most k edges. Conclude that n−1 iterations suffice to find all shortest paths.
▶ Show Full Answer & Explanation
PROOF by induction on k
Base (k=0): Distance[s]=0 = δ₀(s,s). All others: Distance[v]=∞ = δ₀(s,v) (no 0-edge path from s to v≠s). ✓
IH: After k−1 iterations, Distance[v] ≤ δ_{k-1}(s,v) for all v.
IS (k-th iteration): Consider any path P of ≤ k edges from s to v. Let u be the penultimate vertex, with edge (u,v) weight w.
By IH: Distance[u] ≤ δ_{k-1}(s,u) ≤ cost(P up to u).
In iteration k, we relax (u,v): Distance[v] ← min(Distance[v], Distance[u]+w) ≤ Distance[u]+w ≤ cost(P). ✓
Why n−1 suffices: Any shortest simple path has ≤ n−1 edges (never revisit a vertex in an n-vertex graph). So after n−1 iterations, Distance[v] = δ(s,v) for all v. ∎

STABLE MATCHING (KT §1.1)

Gale-Shapley: O(n²). Student-proposing → student-optimal, advisor-pessimal. Check stability: for each non-matched pair, does one prefer their current partner?

ASYMPTOTIC NOTATION (KT §2.2 / CLRS Ch.3)

To prove Θ: show c₁·g(n) ≤ f(n) ≤ c₂·g(n) for n ≥ n₀. Use limit comparison. Key: log^k n = o(n^ε) for any ε>0. Counterexample for claim (c): f=2, g=1+1/n.

GRAPH TRAVERSAL (KT §3 / CLRS Ch.22)

BFS: queue, O(V+E) adj-list, shortest paths (unweighted). DFS: stack/recursion, O(V+E) adj-list, pre/post timestamps, cycle detection (back edge). Edge types: Tree/Forward → containment; Back → ancestor (cycle!); Cross → disjoint intervals.

DAGs & TOPOLOGICAL SORT (KT §3.5)

Every DAG has ≥1 source (in-degree 0 vertex). Remove sources repeatedly. Longest path in DAG = min semesters − 1. DP in topological order: LP[v] = max(LP[u]+1) over incoming edges.

DIVIDE & CONQUER / MASTER THEOREM (KT §5 / CLRS §4)

Case 1: f(n)=O(n^{log_b a − ε}) → T=Θ(n^{log_b a}). Case 2: f(n)=Θ(n^{log_b a}) → T=Θ(n^{log_b a}·log n). Case 3: f(n)=Ω(n^{log_b a + ε}) + regularity → T=Θ(f(n)). Non-applicable: log factor gap → extended MT adds one log.

DYNAMIC PROGRAMMING (KT §6 / CLRS Ch.15)

Identify subproblems. Write recurrence. Compute in order so sub-answers are ready. Floyd-Warshall: W^k[i][j]=min(W^{k-1}[i][j], W^{k-1}[i][k]+W^{k-1}[k][j]). O(n³) time, O(n²) space.

GREEDY (CLRS §16)

Activity selection: sort by finish time, pick earliest compatible. Dijkstra's greedy: settle min-Distance vertex. Correct for non-negative weights (exchange argument). Fails with negative weights.

MST (CLRS Ch.23)

Min connected spanning subgraph with positive weights = tree. Proof: any cycle → remove edge → cheaper + still connected → contradiction.

SHORTEST PATHS (CLRS Ch.24–25)

Dijkstra: O(n²) naive, O((V+E)log V) heap, no negatives. Bellman-Ford: O(VE), negatives OK, n−1 iterations. Floyd-Warshall: O(V³), all pairs, negatives OK. n×Dijkstra heap beats F-W for sparse graphs.

NP-COMPLETENESS (KT §8 / CLRS Ch.34)

IS ↔ VC: S is IS ⟺ V−S is VC. Sizes: |IS| + |VC| = n. VC ≤_p SC: Universe=E, S_v={edges incident to v}, same k. 3-SAT ≤_p IS: triangle gadget, k=clauses. Reduction proofs need BOTH directions of iff!
