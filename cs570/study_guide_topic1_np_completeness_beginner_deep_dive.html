🧗 Warm-up Glossary — Vocabulary You Need Before §1

NP-completeness looks intimidating mostly because it uses precise technical vocabulary that CS courses often assume you already know. None of these concepts are actually hard — they just need to be introduced before we use them. If you already know all of these, skip to §1.

What is a "decision problem"? 📄 p. 249

A decision problem is a computational problem whose answer is simply yes or no. For example: "Does this graph have an independent set of size at least 4?" — yes or no. Compare this to an optimization problem: "What is the largest independent set in this graph?" — that asks for a number.

Why do we care about the yes/no version? Because NP-completeness theory is defined for decision problems. In practice, if the decision version is hard, the optimization version is at least as hard.

What is "polynomial time"? 📄 p. 249

An algorithm runs in polynomial time if its running time on an input of size n is at most O(nk) for some fixed constant k. Examples: O(n), O(n²), O(n³) are all polynomial. In contrast, O(2n) is exponential — it grows astronomically fast.

Concrete example: For n = 100: n² = 10,000 (fine for a computer); 2100 ≈ 1030 (more operations than atoms in the Earth). This is why polynomial algorithms are called "tractable" and exponential algorithms are called "infeasible" for large inputs.

What is "P" (the class of problems)? 📄 p. 249

P is the class of all decision problems that can be solved in polynomial time. Examples: sorting (can you sort this array? yes, always — trivially a decision problem), binary search, shortest paths. If a problem is in P, we have an efficient algorithm for it.

What is "NP"? 📄 p. 249

NP does not stand for "Not Polynomial." It stands for Nondeterministic Polynomial. NP is the class of all decision problems for which a yes-answer can be verified in polynomial time, given a short "certificate" (witness).

Example: Independent Set is in NP. If someone gives you a set S of vertices and claims "S is an independent set of size k," you can check this in polynomial time: just scan all pairs of vertices in S and verify no edge connects them. That checking process is polynomial. Whether you can find S efficiently is a different — and open — question.

⚠️ Classic Misconception "NP means no polynomial solution exists." FALSE. NP means polynomial verification is possible. Whether NP problems can be solved in polynomial time is the famous P vs NP question — unsolved since 1971. 📄 p. 249

What is a "certificate" (or witness)? 📄 p. 249

A certificate is a short piece of information that, if correct, proves a yes-answer. For the Independent Set problem of size k: the certificate is the set S itself (listing k vertex names). For 3-SAT: the certificate is a truth assignment to all variables. Verifying the certificate takes polynomial time.

What is a graph? (Remind me!) 📄 p. 105

A graph G = (V, E) has two components: V is a set of vertices (also called nodes — draw them as circles) and E is a set of edges (draw them as lines connecting circles). An edge (u, v) connects vertex u to vertex v. 

  Example graph with 4 vertices and 4 edges:

       A ——— B
       |     |
       C ——— D

  V = {A, B, C, D}
  E = {(A,B), (A,C), (B,D), (C,D)}

  Vertex B is "adjacent to" A and D (they share an edge).
  Vertex A has degree 2 (two edges touch it).
SOURCE: 📄 p. 105

What does "complement of a set" mean? 📄 pp. 261–263

If S is a subset of V (the vertex set), then V − S (also written V\S) is the complement of S: all vertices NOT in S.

Example: V = {A, B, C, D}, S = {A, D}. Then V − S = {B, C}. Every vertex is either in S or in V − S, never in both. Together they cover all of V.

What is a "polynomial-time reduction"? 📄 p. 253

A reduction from problem X to problem Y (written X ≤p Y) is an algorithm that converts any instance of X into an instance of Y, such that: the answer to the X-instance is YES if and only if the answer to the converted Y-instance is YES. The conversion itself must run in polynomial time.

Think of it as a universal adapter plug: if your plug (X) doesn't fit the socket (solver for Y), the adapter converts it — and any solution from the Y-solver is guaranteed to solve the original X-problem.

What is a "satisfying assignment"? 📄 p. 274

In propositional logic, a satisfying assignment is an assignment of TRUE/FALSE to variables that makes an entire formula evaluate to TRUE.

Example: Formula: (x1 OR x2) AND (NOT x1 OR x3). Assignment x1 = TRUE, x2 = FALSE, x3 = TRUE: both clauses are TRUE, so this is a satisfying assignment.

§1 — What & Why, in Plain English

📘 LECTURE PDF COVERAGE: pp. 249–280 (Lectures 25–27)

1.1 The fundamental problem — why do some problems resist efficient algorithms? 📄 p. 249

Imagine you're a hospital administrator trying to schedule security guards. Your hospital has many corridors, and every corridor must have a guard at one of its two endpoints. You want to hire the minimum number of guards while ensuring every corridor is watched. This is the Vertex Cover problem — and despite decades of research, nobody has found a fast (polynomial-time) algorithm for the general version. 📄 p. 261

Now consider a completely different-looking problem: you're a social network analyst looking for the largest group of people where nobody in the group knows anyone else in the group. This is the Independent Set problem. Strikingly, it is the same problem in disguise as the guard-scheduling problem — a deep connection we will prove rigorously in §2. 📄 pp. 261–263

Here is the central question of NP-completeness theory: Why is it so hard to design efficient algorithms for these problems? Are they all secretly the same problem? Can proving that one is hard prove that all of them are hard? The theory of NP-completeness answers all three questions with a unified framework. 📄 pp. 249–254

P (Polynomial time): Problems solvable efficiently. We have fast algorithms. Examples: sorting, binary search, BFS, shortest paths with non-negative weights. 📄 p. 249

NP (Nondeterministic Polynomial): Problems where a proposed solution can be verified efficiently. Every problem in P is also in NP (solving is harder than verifying, so if you can solve it, you can certainly verify it). But NP is believed to contain problems NOT in P. 📄 p. 249

NP-complete: The "hardest" problems in NP. A problem Q is NP-complete if: (1) Q is in NP, AND (2) every problem in NP reduces to Q in polynomial time. If any NP-complete problem is in P, then P = NP (every NP problem becomes easy). 📄 p. 249

1.3 Reductions as universal adapter plugs — a second framing 📄 pp. 253–254

Suppose you have a mysterious machine that can solve any instance of problem Y in polynomial time — a "Y-oracle." Now you want to solve problem X, but you only have the Y-oracle. Can you still solve X?

Yes, if you can reduce X to Y. A polynomial-time reduction from X to Y is an algorithm that: (1) takes any instance of X, (2) converts it in polynomial time into an instance of Y, such that (3) the Y-oracle's answer to the converted instance gives you the answer to the original X-instance. 

  ┌──────────────┐    Reduction     ┌──────────────┐    Y-oracle    ┌──────────────┐
  │  Instance    │  (polynomial     │  Instance    │   (fast!!)     │   YES / NO   │
  │  of X        │────────────────▶│  of Y        │───────────────▶│  answer for X│
  └──────────────┘  time algorithm) └──────────────┘                └──────────────┘

  Notation:  X ≤p Y  means "X reduces to Y"
             (X is no harder than Y — or equivalently, Y is at least as hard as X)
SOURCE: 📄 p. 253

Critical implication: if X ≤p Y and Y can be solved in polynomial time, then X can also be solved in polynomial time. Contrapositive: if X CANNOT be solved in polynomial time, then neither can Y. This is how we propagate hardness. 📄 p. 254

📌 Key Insight "X ≤p Y" is read "X reduces to Y" — meaning Y is at least as hard as X. If Y were easy, X would be easy too (just use the reduction). Since X is hard (NP-complete), Y must also be hard. DO NOT confuse the direction: X ≤p Y does NOT mean X is harder than Y. 📄 p. 253

1.4 Technical definitions (using lecture notation) 📄 pp. 261–266

Independent Set (IS): Given a graph G = (V, E) and an integer k, does G have an independent set of size k? An independent set is a subset S ⊆ V such that for all u, v ∈ S, the edge (u, v) ∉ E. In plain English: no two vertices in S share an edge. 📄 p. 261

Vertex Cover (VC): Given a graph G = (V, E) and an integer k, does G have a vertex cover of size k? A vertex cover is a subset S ⊆ V such that for every edge (u, v) ∈ E, at least one of u or v is in S. In plain English: every edge has at least one endpoint in S. 📄 p. 261

Set Cover (SC): Given a universe U of elements, a collection of subsets S₁, S₂, ..., Sm of U, and an integer k, are there k subsets whose union is U? In plain English: can k groups together cover all elements? 📄 p. 265

3-SAT: Given a propositional formula in Conjunctive Normal Form (CNF) where every clause has exactly 3 literals, is there a truth assignment satisfying the formula? 📄 p. 274

⚠️ Common Point of Confusion — Know This Before §2 Students often confuse reduction direction with hardness direction. When we write VC ≤p SC, we mean "VC reduces to SC" — which tells us SC is AT LEAST AS HARD as VC. If VC is NP-complete, then SC is NP-hard. The direction of the "≤" arrow says which problem we're converting FROM (the left side) to (the right side). The right-side problem ends up being harder (or equally hard). 📄 p. 254

2a Independent Set ↔ Vertex Cover: The Complementarity Theorem Lecture PDF: pp. 261–263 · Assignment 5 Q2 📋 A5 Q2

Intuition: Guards and non-guards 📄 p. 261

Imagine a graph where each vertex is a person and each edge is a "conflict" between two people. We want to find the largest group of people with no conflicts among them — that's Independent Set. Now flip the question: we want the smallest group of people who touch every conflict — that's Vertex Cover.

These two problems are mirror images. The group of "non-selected" people in the Vertex Cover are exactly the Independent Set, and vice versa. Let's make this precise. 📄 pp. 261–263

Mechanism: The complementarity theorem 📄 pp. 262–263

THEOREM: S is an Independent Set of size k ⟺ V − S is a Vertex Cover of size n − k.
We prove both directions separately (this is mandatory for any "iff" statement).

Worked Example: Concrete 4-vertex graph 📄 pp. 261–263

  Our working graph G = (V, E):

       1 ——— 2
       |     |
       3 ——— 4

  V = {1, 2, 3, 4}
  E = {(1,2), (1,3), (2,4), (3,4)}   (4 edges total)

  Let's try S = {1, 4}.
  Check if S is Independent Set: are there edges within S?
    Edge (1,4)? NOT in E. ✓
  So S = {1,4} IS an independent set of size 2.

  Compute V − S = {2, 3}.
  Claim: V − S = {2, 3} is a Vertex Cover.
  Check every edge:
    (1,2): vertex 2 ∈ {2,3}? YES ✓
    (1,3): vertex 3 ∈ {2,3}? YES ✓
    (2,4): vertex 2 ∈ {2,3}? YES ✓
    (3,4): vertex 3 ∈ {2,3}? YES ✓
  All 4 edges are covered. V − S = {2,3} IS a vertex cover of size 2. ✓
SOURCE: 📄 pp. 261–263

Forward direction proof: S is IS → V−S is VC 📄 pp. 262–263

Step 1: Assume S is an independent set.
This means: for any two vertices u, v both in S, the edge (u,v) is NOT in E. No edge "stays within" S. 📄 p. 261
Step 2: Take any edge (u, v) ∈ E.
We need to show that at least one of u, v is in V−S (i.e., at least one is NOT in S).
Step 3: Can BOTH u and v be in S?
No! If both u ∈ S and v ∈ S, then the edge (u,v) would exist within S — but S is an independent set, so no such edge exists. Contradiction. 📄 p. 262
Step 4: Therefore, at least one of u, v is NOT in S.
Equivalently, at least one of u, v IS in V−S. So V−S covers the edge (u,v).
Step 5: Conclusion
Since (u,v) was an arbitrary edge, V−S covers ALL edges. So V−S is a vertex cover. ✓ 📄 p. 262

Backward direction proof: V−S is VC → S is IS 📄 pp. 262–263

Step 1: Assume V−S is a vertex cover.
This means: for every edge (u,v) ∈ E, at least one of u, v is in V−S (i.e., NOT in S). 📄 p. 261
Step 2: Suppose for contradiction that S is NOT independent.
Then there exist u, v ∈ S with (u,v) ∈ E — an edge entirely within S.
Step 3: But if u ∈ S and v ∈ S, then u ∉ V−S and v ∉ V−S.
Edge (u,v) has NEITHER endpoint in V−S. So V−S does NOT cover this edge. 📄 p. 262
Step 4: Contradiction.
We assumed V−S covers ALL edges. But we just found an edge it doesn't cover. This is impossible. 📄 p. 263
Step 5: Conclusion
Our assumption (S is not independent) was wrong. Therefore S IS an independent set. ✓ 📄 p. 263
⚠️ Most Common Grading Mistake On Assignment 5 and likely on the final: many students prove only ONE direction of the iff and claim they are done. An "iff" proof ALWAYS requires two separate sub-proofs: (→) and (←). Each sub-proof gets partial credit independently. Stating "the reverse direction is similar" is NOT acceptable — you must write it out. 📄 pp. 262–263

2b Vertex Cover ≤p Set Cover: A Full Reduction Walkthrough Lecture PDF: pp. 265–266 · Assignment 5 Q3 📋 A5 Q3

Intuition: Edges as elements, vertices as sets 📄 p. 265

In Vertex Cover, we select a minimum set of vertices such that every edge is covered. In Set Cover, we select a minimum number of subsets such that every element is covered.

The clever insight is to map one onto the other: make the edges become the elements (our universe U), and make each vertex become a subset containing all edges incident to it. Now "covering every edge" becomes "covering every element" — exactly Set Cover. 📄 pp. 265–266

Mechanism: The formal construction 📄 pp. 265–266

Given a Vertex Cover instance (G = (V, E), k), construct a Set Cover instance (U, S, k) as follows:

Construction Step 1: Set the universe U = E
Each element of U is an edge of G. We want to "cover" all edges.
Construction Step 2: For each vertex v ∈ V, create a subset S_v
S_v = { e ∈ E : e is incident to v } — the set of all edges that touch vertex v.
Construction Step 3: Keep the parameter k the same
We ask: "Is there a set cover of size k?" with the same k as in the VC instance.
Step 4: The conversion is the polynomial
We iterate over all vertices and edges exactly once: O(|V| + |E|) time — definitely polynomial. 📄 p. 265

Worked Example: VC ≤p SC on our 4-vertex graph 📄 pp. 265–266 📋 A5 Q3

  Original VC instance: G with 4 vertices, 4 edges, k = 2.

       1 ——(e1)—— 2
       |           |
      (e2)        (e3)
       |           |
       3 ——(e4)—— 4

  Edges: e1=(1,2), e2=(1,3), e3=(2,4), e4=(3,4)

  ─── CONSTRUCTION ───────────────────────────────────────────────

  Universe U = E = {e1, e2, e3, e4}     (the 4 edges are our elements)

  Subset for vertex 1:  S_1 = {e1, e2}  (edges incident to 1: (1,2),(1,3))
  Subset for vertex 2:  S_2 = {e1, e3}  (edges incident to 2: (1,2),(2,4))
  Subset for vertex 3:  S_3 = {e2, e4}  (edges incident to 3: (1,3),(3,4))
  Subset for vertex 4:  S_4 = {e3, e4}  (edges incident to 4: (2,4),(3,4))

  SC question: Can we cover U = {e1, e2, e3, e4} using 2 subsets?

  ─── VERIFY THE REDUCTION ────────────────────────────────────────

  VC answer: YES — vertex cover {1,4} of size 2 exists.
    (Check: edge e1=(1,2) covered by 1 ✓; e2=(1,3) covered by 1 ✓;
            e3=(2,4) covered by 4 ✓; e4=(3,4) covered by 4 ✓)

  Corresponding SC answer: Select subsets {S_1, S_4}.
    S_1 ∪ S_4 = {e1, e2} ∪ {e3, e4} = {e1, e2, e3, e4} = U ✓
  So the SC answer is also YES. ✓

  ─── REVERSE CHECK ───────────────────────────────────────────────

  Suppose the SC oracle returns {S_2, S_3}.
    S_2 ∪ S_3 = {e1, e3} ∪ {e2, e4} = {e1, e2, e3, e4} = U ✓
  So SC says YES with {S_2, S_3}.
  Map back to VC: select vertices {2, 3}.
  Verify VC: e1=(1,2) covered by 2 ✓; e2=(1,3) covered by 3 ✓;
             e3=(2,4) covered by 2 ✓; e4=(3,4) covered by 3 ✓. ✓
  VC answer is also YES. ✓

Proof of correctness — BOTH directions 📄 pp. 265–266

Direction (→): If G has a VC of size k, then the SC instance has a set cover of size k.

Assume: VC = {v₁, v₂, ..., v_k} is a vertex cover of G.
Every edge e ∈ E has at least one endpoint in {v₁, ..., v_k}.
Claim: the corresponding subsets {S_{v₁}, ..., S_{v_k}} cover U.
Take any element e ∈ U. Since U = E, e is an edge. Since the VC covers e, at least one endpoint of e, say v_i, is in the VC. Then e ∈ S_{v_i} (by construction: S_{v_i} contains all edges incident to v_i). So e is in the union of our chosen subsets. 📄 p. 265
Conclusion: all elements of U are covered, so k subsets suffice. ✓

Direction (←): If the SC instance has a set cover of size k, then G has a VC of size k.

Assume: {S_{v₁}, ..., S_{v_k}} is a set cover of U = E.
Their union contains all edges: for every e ∈ E, some S_{v_i} contains e.
Claim: the vertices {v₁, ..., v_k} form a vertex cover of G.
Take any edge e = (u, w) ∈ E. Since the SC covers it, e ∈ S_{v_i} for some i. By construction, S_{v_i} = {all edges incident to v_i}. So e is incident to v_i. That means v_i is an endpoint of e — i.e., v_i = u or v_i = w. So e is covered by vertex v_i. 📄 p. 266
Conclusion: every edge is covered by some v_i, so {v₁,...,v_k} is a vertex cover. ✓
2c 3-SAT ≤p Independent Set: The Triangle Gadget Lecture PDF: p. 275 (Lecture 27)

Intuition: Variables map to vertices, clauses map to triangles 📄 p. 275

This reduction converts a 3-SAT formula into a graph such that the formula is satisfiable if and only if the graph has an independent set of size equal to the number of clauses.

The key idea is a gadget: for each clause (with 3 literals), create a triangle (3-cycle) of vertices, one vertex per literal. To pick an independent set, we can select at most one vertex per triangle (because all three are mutually connected). The selected vertex represents the literal we set to TRUE to satisfy that clause. 📄 p. 275

Mechanism: The construction 📄 p. 275

Given 3-SAT instance Φ with m clauses and variables x₁,...,x_n:

Step 1: Create m triangles (one per clause)
For clause C_j = (ℓ_{j,1} OR ℓ_{j,2} OR ℓ_{j,3}), create vertices v_{j,1}, v_{j,2}, v_{j,3}. Add edges (v_{j,1},v_{j,2}), (v_{j,2},v_{j,3}), (v_{j,1},v_{j,3}) — forming a triangle.
Step 2: Add "conflict edges" between complementary literals
If vertex v_{j,r} represents literal ℓ and vertex v_{j',r'} represents ¬ℓ (the negation), add edge (v_{j,r}, v_{j',r'}). This prevents picking both a literal and its negation simultaneously.
Step 3: Set k = m (number of clauses)
Ask: "Is there an IS of size m in this graph?"

2d Proof Template: The "Both Directions" Skeleton for Every Reduction Lecture PDF: pp. 261–266 (Lectures 26–27)

Why every reduction proof must prove both directions 📄 pp. 262–266

A reduction proves an "if and only if" (iff) statement. When we construct the SC instance from a VC instance, we need to prove: (→) "If VC has a YES answer, then SC has a YES answer." (←) "If SC has a YES answer, then VC has a YES answer."

Together, these say the two instances are equivalent — they always give the same YES/NO answer. This is what makes the reduction valid. 📄 p. 266

Assignment Replay: A5 Worked Solutions 📋 A5

📋 A5 Q1 Give formal definitions of IS, VC, SC with examples.

▶ Key Points in the Solution
  • IS: S ⊆ V such that for all u, v ∈ S, (u,v) ∉ E. The k refers to |S| ≥ k (we want a large IS). Example: {B, C, E, G} in the lecture's 7-vertex graph. 📄 p. 261
  • VC: S ⊆ V such that for all (u,v) ∈ E, u ∈ S or v ∈ S (or both). We want the smallest such S, so k refers to |S| ≤ k. Example: {A, D, F}. 📄 p. 261
  • SC: Universe U, collection of subsets {S₁,...,Sm} with each S_i ⊆ U. We want k subsets whose union = U. Example: U = {1,...,7}, subsets as in the software example. 📄 p. 265
  • Grading criterion (100%): Must include quantifier structure (∀ vertices in S, no edge exists; ∀ edges, ∃ endpoint in S), size parameter k, and a concrete example. 📄 pp. 261–265
  • Exam variant: "Is the following set S a valid IS/VC? Verify." — Verify ALL edges, not a sample. 📄 pp. 261–265
  • Deeper insight: The IS problem maximizes (find the LARGEST conflict-free group), while VC minimizes (find the SMALLEST covering set). This asymmetry is why their complementary sizes add up to n. 📄 pp. 261–263

📋 A5 Q2 Prove INDEPENDENT-SET =p VERTEX-COVER (both directions).

▶ Key Points in the Solution
  • The theorem: S is an IS of size k iff V−S is a VC of size n−k. This is equivalence (=p), requiring two reductions: IS ≤p VC and VC ≤p IS. 📄 pp. 262–263
  • Forward (→): If S is an IS, take any edge (u,v). Since S is independent, u and v cannot both be in S. So at least one is in V−S → V−S covers (u,v). Since edge was arbitrary, V−S is a VC. 📄 p. 262
  • Backward (←): If V−S is a VC, suppose S has an edge (u,v). Then u ∈ S and v ∈ S, so u ∉ V−S and v ∉ V−S — edge (u,v) is not covered. Contradiction with V−S being a VC. So S has no edges — S is an IS. 📄 p. 263
  • Why IS ≤p VC holds: Given IS instance (G, k), produce VC instance (G, n−k). Same graph, flip the parameter. O(1) construction time. 📄 pp. 262–263
  • Why VC ≤p IS holds: Symmetric — given VC instance (G, k), produce IS instance (G, n−k). 📄 pp. 262–263
  • Grading criterion (100%): State theorem clearly; write both directions separately with all proof steps; check the size arithmetic (n−k); state the reduction constructions. 📄 pp. 262–263
  • Exam variant: "Given a VC of size k, find an IS. What is its size?" Answer: V−VC has size n−k and is the IS. 📄 pp. 262–263

📋 A5 Q3 Prove VERTEX-COVER ≤p SET-COVER (full reduction).

▶ Key Points in the Solution
  • Construction: U = E (universe is the set of edges). For each vertex v, S_v = {e ∈ E : v is an endpoint of e}. Keep k. 📄 p. 265
  • (→) direction: VC {v₁,...,v_k} → SC {S_{v₁},...,S_{v_k}}. For any edge e, VC covers it → some v_i is an endpoint → e ∈ S_{v_i} → e is in the union → SC is valid. 📄 pp. 265–266
  • (←) direction: SC {S_{v₁},...,S_{v_k}} covers all elements → for any edge e, e ∈ S_{v_i} for some i → v_i is incident to e → {v₁,...,v_k} covers all edges → valid VC. 📄 p. 266
  • Runtime: O(|V| + |E|) — iterate over all vertices and their incident edges once to build all S_v. 📄 p. 265
  • Why the construction is clever: "A vertex covers an edge" in VC ↔ "an edge (as an element) belongs to the vertex's subset" in SC. The semantic match is perfect. 📄 pp. 265–266
  • Grading criterion (100%): Construction stated formally with U, sets, and k; both iff-directions written out with all steps; runtime mentioned explicitly. 📄 pp. 265–266
  • Exam variant: "What is the SC instance for this specific graph? Find a set cover and map it back to a VC." — Trace the construction as in Q9 of §4. 📄 pp. 265–266

Misconception Cards 📄 pp. 249–280

❌ Misconception: "NP-complete problems have no polynomial-time algorithm."

✅ Reality: NP-complete problems have no known polynomial-time algorithm. The P vs NP question is unsolved. NP-complete means "as hard as the hardest problems in NP," not "provably exponential." 📄 p. 249

❌ Misconception: "X ≤p Y means X is harder than Y."

✅ Reality: X ≤p Y means Y is at least as hard as X. If you can solve Y, you can solve X. The left-side problem is being reduced to the right-side problem — the right side must be powerful enough to handle the left. 📄 p. 253

❌ Misconception: "An independent set and a vertex cover are different ways to solve the same problem."

✅ Reality: They ask complementary questions on the SAME graph. IS asks for the largest conflict-free group; VC asks for the smallest covering group. They are different problems that are equivalent in computational difficulty via the complementarity theorem. 📄 pp. 261–263

❌ Misconception: "Proving only one direction of VC ≤p SC is enough."

✅ Reality: A valid reduction requires proving an "iff" — both (→) and (←). One direction shows the construction preserves YES-answers (completeness). The other shows it doesn't create spurious YES-answers (soundness). Both are required for full credit. 📄 pp. 265–266

❌ Misconception: "3-SAT ≤p IS means IS reduces to 3-SAT."

✅ Reality: 3-SAT ≤p IS means 3-SAT reduces TO IS. We convert 3-SAT instances into IS instances. IS must be at least as hard as 3-SAT. Since 3-SAT is NP-complete, IS is NP-hard (and since IS is also in NP, IS is NP-complete). 📄 p. 275

❌ Misconception: "The VC ≤p SC construction sets U = V (the vertex set)."

✅ Reality: U = E (the EDGE set). The elements of the Set Cover universe are the EDGES of the graph, not the vertices. Vertices become the subsets S_v. This confusion is the most common construction error on the exam. 📄 p. 265

❌ Misconception: "A polynomial-time approximation algorithm for VC contradicts NP-hardness."

✅ Reality: NP-hardness says we cannot find the EXACT minimum VC in polynomial time (assuming P ≠ NP). A 2-approximation (finding a VC of size ≤ 2 × OPT) runs in polynomial time and gives a near-optimal answer — this is completely consistent with NP-hardness. The hardness is only about finding the exact optimum. 📄 p. 249

Memory Aids & Mnemonics 📄 pp. 249–280

🧠 Mnemonic 1: IS vs VC — "Inside vs Edge-Guard" IS = "Inside" group — no conflicts WITHIN the group (no edges inside S). VC = "Edge Guard" — every edge has a guard at one end (every edge has an endpoint in S). They are literal complements: IS guards the inside, VC guards the edges from outside. 📄 pp. 261–263
🧠 Mnemonic 2: VC→SC Construction — "Edges become Elements, Vertices become Sets" To reduce VC to SC: Edges ↦ Elements (Universe U = E), Vertices ↦ Subsets (S_v = edges incident to v). Parameter k stays the same. One-line summary: "EE VS k" — Edges to Elements, Vertices to Subsets, k unchanged. 📄 pp. 265–266
🧠 Mnemonic 3: Reduction direction — "X ≤p Y means Y is Your oracle" When you write X ≤p Y: Y is the oracle (solver) you're calling. You transform X-instances so that Y can answer them. Y must be at least as capable as whatever X needs. Hardness flows from left to right: if X is hard, Y must also be hard (to handle X). 📄 p. 253
🧠 Mnemonic 4: Both directions checklist — "CAVE" For every NP reduction proof: Construction (write U, sets, k), Arrow forward (→), Verify backward (←), Efficiency (state runtime). Miss any of CAVE = lose points. 📄 pp. 262–266

WC-1. A classmate says: "Vertex Cover and Independent Set are the same problem — they both work on graphs and deal with vertices." Explain what they got right and what they got wrong. Be specific.

WARM-UP PROOF LABELING 📄 pp. 262–263

WC-2. Below is a proof sketch with labels removed. Identify the role of each numbered step.

  (1) Let S be a subset of V such that for all u,v ∈ S, (u,v) ∉ E.
  (2) Consider any edge (a,b) ∈ E.
  (3) Suppose both a ∈ S and b ∈ S.

(5) Conclusion from contradiction — the assumption was false, so at least one endpoint is outside S. 📄 p. 262

WC-5. Your classmate says: "If we can't find a polynomial-time algorithm for IS after 50 years of trying, that proves IS is not in P." Explain what they got wrong about the logic of NP-completeness.

▶ Show Explanation

Q7. Claim: "A graph G has an independent set of size k if and only if it has a vertex cover of size n − k." Prove both directions of this equivalence AND use it to show that INDEPENDENT-SET =p VERTEX-COVER (i.e., they are computationally equivalent — each reduces to the other in polynomial time).

▶ Show Answer (Full Model Proof)

✅ Both directions of the iff + reduction argument.

Part 1: The complementarity theorem (both directions)

(→) IS of size k → VC of size n−k:
Let S be an IS of size k. For any edge (u,v) ∈ E: if both u,v ∈ S, then (u,v) is an edge within S — contradicting S being independent. So at least one of u,v is in V−S. Hence V−S covers all edges. |V−S| = n−k. ✓ 📄 pp. 261–262
(←) VC of size n−k → IS of size k:
Let T = V−S be a VC of size n−k, so S = V−T has size k. Suppose S has an edge (u,v). Then u ∈ S and v ∈ S, so u ∉ T and v ∉ T — edge (u,v) has no endpoint in T. But T is a VC (covers all edges). Contradiction. So S has no edges — S is an IS of size k. ✓ 📄 pp. 262–263

Part 2: IS ≤p VC reduction 📄 pp. 262–263

Given IS instance (G, k): construct VC instance (G, n−k) (same graph, complementary parameter). By the theorem: IS answer YES iff VC answer YES. Construction is O(1) (just compute n−k). So IS ≤p VC. ✓

Part 3: VC ≤p IS reduction 📄 pp. 262–263

Given VC instance (G, k): construct IS instance (G, n−k) (same graph, complementary parameter). By the theorem: VC answer YES iff IS answer YES. Construction is O(1). So VC ≤p IS. ✓

Conclusion: Since IS ≤p VC and VC ≤p IS, we have IS =p VC — they are computationally equivalent. ∎ 📄 pp. 262–263

  • ☐ Both (→) and (←) directions of the iff theorem proved. 📄 pp. 261–263

Q11. Write pseudocode to verify whether a given subset S of vertices is a valid Independent Set in a graph G = (V, E) given as an adjacency list. Analyze time and space complexity.

▶ Show Answer

✅ Pseudocode:

function isIndependentSet(G, S):
    # G is given as an adjacency list: G[v] = list of neighbors of v
    # S is the candidate independent set (a list of vertices)

    # Convert S to a set for O(1) membership testing
    S_set = toSet(S)

    # Check every vertex u in S
    for u in S:
        # Check all neighbors of u
        for v in G[u]:               # v is a neighbor of u
            if v in S_set:           # neighbor is also in S
                return False         # edge (u,v) is inside S — not independent!

    return True                      # no edge found within S
SOURCE: 📄 pp. 261–263

English Explanation:

  1. Build a fast-lookup set from S (so "v in S?" is O(1)).
  2. For each vertex u in S, scan all its neighbors using the adjacency list.
  3. If any neighbor v is also in S, edge (u,v) is "inside" S — S is NOT an independent set. Return False.
  4. If we finish all neighbors without finding an internal edge: return True.

📝 Complexity Analysis:

Time: Building S_set = O(|S|). Scanning neighbors of all vertices in S = O(Σ_{u∈S} deg(u)) ≤ O(|E|). Total: O(|S| + |E|). 📄 p. 261

Space: S_set = O(|S|); adjacency list already given = O(|V| + |E|). Extra space = O(|S|). 📄 p. 261

Q12. Write pseudocode implementing the VC ≤p SC reduction: given a graph G = (V, E) and integer k, output the Set Cover instance (U, subsets, k). Provide a step-wise English explanation and analyze time and space complexity.

▶ Show Answer

✅ Pseudocode:

function VCtoSC(G, k):
    # G = (V, E) given as adjacency list
    # Returns: (universe U, collection_of_subsets, k)

    # Step 1: Universe = all edges
    U = set()
    for v in G.vertices:
        for w in G.adjacency[v]:
            edge = makeEdge(v, w)   # normalize: (min(v,w), max(v,w))
            U.add(edge)

    # Step 2: For each vertex, build its subset (incident edges)
    subsets = {}
    for v in G.vertices:
        S_v = set()
        for w in G.adjacency[v]:
            edge = makeEdge(v, w)
            S_v.add(edge)
        subsets[v] = S_v

    # Step 3: Return the SC instance
    return (U, subsets, k)     # same k as VC instance
SOURCE: 📄 pp. 265–266

English Explanation:

  1. Iterate over all edges and add each to the universe U. Use a canonical form (min,max) to avoid duplicates in undirected graphs.
  2. For each vertex v, collect all edges incident to v into the subset S_v. This represents "what vertex v can cover."
  3. Return the triple (U, all S_v subsets, k). The SC oracle can now find k subsets covering all edges.

📝 Complexity Analysis:

Time: Building U requires scanning all adjacency lists = O(|V| + |E|). Building subsets = same pass = O(|V| + |E|). Total: O(|V| + |E|) — polynomial. 📄 p. 265

Space: U holds |E| edges = O(|E|). Subsets dictionary holds each edge at most twice (once per endpoint) = O(|E|). Total: O(|V| + |E|). 📄 p. 265

Q13. A greedy heuristic for Vertex Cover works as follows: repeatedly pick an uncovered edge (u,v), add BOTH u and v to the cover, mark all their edges as covered, and repeat until no edges remain. Write pseudocode for this heuristic. Analyze its time complexity. Prove it always finds a VC of size at most 2 × OPT (where OPT is the minimum VC size). Explain why this does NOT contradict NP-hardness of VC.

▶ Show Answer

✅ Pseudocode (2-approximation for Vertex Cover):

function greedyVC(G):
    # G = (V, E) given as adjacency list
    cover = empty set
    covered_edges = empty set

    for each edge (u, v) in E:
        if (u, v) not in covered_edges:
            cover.add(u)
            cover.add(v)
            # Mark all edges incident to u and v as covered
            for w in G.adjacency[u]:
                covered_edges.add(makeEdge(u, w))
            for w in G.adjacency[v]:
                covered_edges.add(makeEdge(v, w))

    return cover     # a vertex cover (not necessarily minimum)
SOURCE: 📄 pp. 261–263 (greedy concept); approximation algorithms context.

English Explanation: Scan edges in any order. When you find an uncovered edge, add BOTH endpoints to the cover (conservative — ensures the edge is covered) and remove all edges touching either endpoint. Repeat until all edges are covered.

📝 Time Complexity: Each edge is examined at most twice (once when scanned, once when marked covered). Marking covered edges of u and v = O(deg(u) + deg(v)) per iteration. Total across all iterations = O(|V| + |E|). O(|V| + |E|). 📄 pp. 261–263

Proof: |cover| ≤ 2 × OPT.

Let M be the set of edges selected by the algorithm (the edges where we chose to add both endpoints). Each such edge is distinct — once we pick an edge, all its neighboring edges get covered and cannot be picked again. So M is a matching (no two edges in M share an endpoint). 📄 p. 261

|cover| = 2|M| (we add 2 vertices per edge in M).

OPT ≥ |M| because any vertex cover must include at least one endpoint from each edge in M. Since edges in M share no endpoints, OPT needs at least |M| distinct vertices to cover them all.

Therefore: |cover| = 2|M| ≤ 2 × OPT. ✓

Why this doesn't contradict NP-hardness: We have an APPROXIMATION algorithm — it's fast (polynomial) but produces an answer that might be up to 2× the optimal. NP-hardness says we cannot find the exact minimum VC in polynomial time (assuming P ≠ NP). Approximate solutions within a constant factor are still possible and very useful in practice. 📄 p. 249
